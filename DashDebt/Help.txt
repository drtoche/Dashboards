First steps after creating a user account on render.com

1. Select "Web Service"
Why: "Static Sites" are only for basic HTML/CSS (no Python). A Web Service allows Render to run your Python interpreter and the Dash server.

Action: Click the "Build and deploy from a Git repository" button inside the Web Service tile.

2. Connect Your GitHub
You will see a list of your GitHub repositories.

Find the project you just saved and click Connect.

3. Fill in the "Form" (The Settings)
Once you click connect, you’ll see the page with the settings we discussed. Here is a quick cheat sheet for those specific fields:

Name: Pick anything (e.g., my-dash-app-2026).

Region: Select Singapore (since it is closest to the Greater Bay Area and will give you the lowest "latency" or lag).

Branch: Leave it as main.

Language: Select Python 3.

Free Plan: Ensure the "Free" instance type is selected ($0/month).

4. The "Build & Start" Commands
Scroll down to find these two specific boxes. This is the most important part:

Build Command: ```bash
pip install -r requirements.txt

Start Command: ```bash
gunicorn app:server

*(Note: If your file is named something else, like `mydash.py`, use `gunicorn mydash:server` instead.)*

5. Final Step: Create
Click Create Web Service at the bottom of the page.

What to watch for now:
A black "Logs" screen will appear. You will see lines of text as Render installs your libraries. It usually takes 2–5 minutes the first time.

Important: If it fails, look at the very bottom of the logs. It will usually say something like ModuleNotFoundError: No module named 'pandas'. This just means you forgot to add that library to your requirements.txt file on GitHub.
